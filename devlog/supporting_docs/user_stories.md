## U1. Stealth and Movement

| ID | User Story | Priority |
|---|---|---|
| U1.1 | As a player, I want responsive movement so I can position precisely to avoid enemy vision and alert zones. | High |
| U1.2 | As a player, I want walls, doors, and solid obstacles to block movement so navigation feels consistent. | High |
| U1.3 | As a player, I want obstacles and corners to provide cover so I can avoid line of sight strategically. | High |
| U1.4 | As a player, I want to switch between walking and sprinting so that I can trade safety for speed when escaping danger. | High |
| U1.5 | As a player, I want sprinting to generate visible/audible noise feedback so I understand when enemies may investigate. | High |
| U1.6 | As a player, I want guards to lose me when I successfully break line-of-sight so that stealth feels skill-based. | High |
| U1.7 | As a player, I want to fail only when a guard reaches me in a chase so that mistakes feel fair and understandable. | High |
| U1.8 | As a player, I want to preview nearby danger, using a panning camera, so I can scout safely before advancing. | Medium |

## U2. Enemy Visibility and Feedback

| ID | User Story | Priority |
|---|---|---|
| U2.1 | As a player, I want to see the enemy field of view (FOV) so I can make informed stealth decisions. | High |
| U2.2 | As a player, I want walls, closed doors, and obstacles to block enemy vision so I’m not unfairly detected. | High |
| U2.3 | As a player, I want a detection bar so I can quickly see if an enemy has detected me. | High |
| U2.4 | As a player, I want immediate spotted feedback, sound/visual, so I know exactly when I’m detected. | High |
| U2.5 | As a player, I want the detection bar to decay over time when I’m out of sight so I can recover through good play. | High |
| U2.6 | As a player, I want clear transition cues when a guard changes states so I can respond immediately. | High |
| U2.7 | As a player, I want guards to look or behave differently when patrolling, investigating, searching, or chasing so I can make tactical decisions. | High |

## U3. Rooms, Doors, and Progression

| ID | User Story | Priority |
|---|---|---|
| U3.1 | As a player, I want interconnected rooms and doors so the level layout feels coherent. | High |
| U3.2 | As a player, I want locked doors to clearly indicate they’re locked so I understand why I can’t pass. | High |
| U3.3 | As a player, I want doors to unlock when I have the key so I can progress after completing objectives. | High |
| U3.4 | As a player, I want to have exits clearly identifiable so that I know where to go to complete the level. | High |
| U3.5 | As a player, I want a level path to be completable in 2-5 minutes so that the experience feels polished and replayable. | Medium |
| U3.6 | As a player, I want a minimap to locate myself in the map. | Low |
| U3.7 | As a player, I want rooms I’ve entered to remain visible so I can plan routes using memory. | Medium |
| U3.8 | As an explorer, I want optional side areas so I feel rewarded for careful searching. | Medium |

## U4. Keys and Interactables

| ID | User Story | Priority |
|---|---|---|
| U4.1 | As a player, I want to pick up keys when I reach them so collecting feels responsive. | High |
| U4.2 | As a player, I want keys to be visible and distinct from the background so exploration is readable. | High |
| U4.3 | As a player, I want a HUD indicator showing keys collected so I can confirm if I can unlock a door. | High |
| U4.4 | As a player, I want clear prompts for interactable objects so that I know when I can take an action. | High |
| U4.5 | As a player, I want light toggling to influence stealth conditions so I can create tactical opportunities. | High |
| U4.6 | As a casual player, I want simple, consistent interaction inputs so controls are easy to remember. | Medium |
| U4.7 | As a casual player, I want keys to communicate their purpose clearly so I immediately know what they are for. | Medium |

## U5. Challenge and Learning Curve

| ID | User Story | Priority |
|---|---|---|
| U5.1 | As a player, I want an early room that teaches line of sight and cover so I can learn by doing. | Medium |
| U5.2 | As a player, I want a short tutorial that explains inputs so I know how to interact with the game. | Medium |
| U5.3 | As a player, I want patrol patterns to be predictable so success depends on observation. | High |
| U5.4 | As a player, I want guards to investigate suspicious events so my actions have consequences. | High |
| U5.5 | As a player, I want guards to search after losing sight and eventually de-escalate if they can’t find me so escapes are tense but fair. | High |
| U5.6 | As a casual player, I want difficulty to ramp room by room so I can build confidence before harder scenarios. | High |

## U6. Portal Mechanics

| ID | User Story | Priority |
|---|---|---|
| U6.1 | As a player, I want to place two portals on valid surfaces so I can create traversal shortcuts. | High |
| U6.2 | As a player, I want entering one portal to move me to the paired portal so portal behavior is reliable. | High |
| U6.3 | As a player, I want placing a third portal to remove the oldest portal so portal placement is predictable. | High |
| U6.4 | As a player, I want clear feedback when trying to place a portal on an invalid surface so placement rules are understandable. | Medium |
| U6.5 | As a player, I want enemies to ignore portals so stealth behavior stays consistent. | Medium |

## U7. Inventory and Items

| ID | User Story | Priority |
|---|---|---|
| U7.1 | As a player, I want three inventory slots shown in the HUD so I can track my available item space and current items. | High |
| U7.2 | As a player, I want item pickups to fill empty slots from left to right so inventory behavior is predictable. | High |
| U7.3 | As a player, I want a clear “inventory full” signal when no slot is available so I understand why item pickup fails. | Low |
| U7.4 | As a player, I want to use items from specific slots so I can react quickly in stealth scenarios. | Medium |
| U7.5 | As a player, I want to free a slot by using an item before picking up a new one so inventory management is meaningful. | High |
| U7.6 | As an explorer, I want optional nonessential items with distinct effects so I can experiment with different strategies. | Low |

## U8. Game Flow and Feedback

| ID | User Story | Priority |
|---|---|---|
